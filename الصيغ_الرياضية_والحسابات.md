# 📐 الصيغ الرياضية والمعادلات (Mathematical Formulas & Equations)

## تحديد الرموز
```
P_var              = التأخير المتوقع (Expected Delay) - بالأيام
base_delay         = التأخير الأساسي (Base Delay) - بالأيام
weather_delay      = تأخير الطقس (Weather Delay) - بالأيام
sel_labor          = كفاءة العمالة (Labor Efficiency) - من 0 إلى 1
sel_days           = مدة المرحلة (Phase Duration) - بالأيام
sel_region         = المنطقة الجغرافية (Region)
risk_percentage    = معدل الخطر (Risk Percentage) - %
cost_overrun_val   = تجاوز التكلفة (Cost Overrun) - %
```

---

## ✅ المعادلات الأساسية

### 1️⃣ حساب التأخير الأساسي (Base Delay Formula)

```
base_delay = (1.0 - sel_labor) × (sel_days × 0.4)
```

**المتغيرات:**
- `sel_labor`: معامل كفاءة العمالة (0 ≤ sel_labor ≤ 1)
- `sel_days`: عدد الأيام المخطط للمرحلة
- `0.4`: معامل الأثر (Impact Factor)

**الشرح:**
- كلما انخفضت كفاءة العمالة (أقرب إلى 0)، زاد التأخير الأساسي
- كلما زادت مدة المرحلة، زاد التأخير الأساسي
- العلاقة خطية مع `sel_days` وعكسية مع `sel_labor`

**مثال حسابي:**
```
إذا: sel_labor = 0.7 (70% كفاءة)
     sel_days = 50 يوم

base_delay = (1.0 - 0.7) × (50 × 0.4)
           = 0.3 × 20
           = 6 أيام
```

---

### 2️⃣ حساب تأخير الطقس (Weather Delay Formula)

```
إذا sel_region في ["قطاع الرياض", "نيوم", "Riyadh", "NEOM"]:
    weather_delay = sel_days × 0.15
وإلا:
    weather_delay = 0
```

**الشرح:**
- المناطق الصحراوية (الرياض ونيوم) قد تعاني من:
  - درجات حرارة عالية جداً (40-50°C)
  - العواصف الرملية والأتربة
  - مما يسبب توقفات العمل المتكررة
  
- معامل التأثير `0.15` يعني 15% من مدة المرحلة

**مثال حسابي:**
```
إذا: sel_region = "قطاع الرياض"
     sel_days = 50 يوم

weather_delay = 50 × 0.15 = 7.5 أيام

لكن إذا: sel_region = "جدة" (منطقة ساحلية معتدلة)
weather_delay = 0 (لا يوجد تأثير طقسي مباشر)
```

---

### 3️⃣ حساب إجمالي التأخير المتوقع (Total Expected Delay)

```
P_var = round(base_delay + weather_delay, 1)
```

**الشرح:**
- جمع التأخير الأساسي وتأخير الطقس
- تقريب النتيجة إلى منزلة عشرية واحدة

**مثال متكامل:**
```
المنطقة: قطاع الرياض
كفاءة العمالة: 70%
مدة المرحلة: 50 يوم

الخطوة 1: base_delay = (1 - 0.7) × (50 × 0.4) = 0.3 × 20 = 6 أيام
الخطوة 2: weather_delay = 50 × 0.15 = 7.5 أيام
الخطوة 3: P_var = round(6 + 7.5, 1) = 13.5 أيام

النتيجة: التأخير المتوقع = 13.5 يوم من أصل 50 يوم (تأخير بنسبة 27%)
```

## 📊 مؤشرات الأداء والخطر

### 4️⃣ حساب معدل الخطر (Risk Percentage)

```
risk_percentage = min(int((P_var ÷ sel_days) × 100) + 15, 95)
```

**الشرح التفصيلي:**
1. `P_var ÷ sel_days`: نسبة التأخير إلى المدة الكلية (كنسبة عشرية)
2. `× 100`: تحويل إلى نسبة مئوية
3. `int(...)`: تحويل إلى عدد صحيح
4. `+ 15`: إضافة هامش أمان (15%)
5. `min(..., 95)`: حد أقصى 95% (تجنب التشاؤم الشديد)

**جدول التفسير:**
| معدل الخطر | التقييم | الإجراء المطلوب |
|-----------|---------|---------------|
| 0-20% | ✅ خطر منخفض جداً | مراقبة روتينية |
| 21-40% | 🟢 خطر منخفض | مراقبة دورية |
| 41-60% | 🟡 خطر متوسط | تدخل استباقي |
| 61-80% | 🟠 خطر عالي | تدخل فورى |
| 81-95% | 🔴 خطر شديد جداً | تدخل طارئ |

**أمثلة عددية:**
```
مثال 1: P_var = 13.5 أيام، sel_days = 50 أيام
risk_percentage = min(int((13.5 ÷ 50) × 100) + 15, 95)
                = min(int(27) + 15, 95)
                = min(42, 95)
                = 42% (خطر متوسط)

مثال 2: P_var = 35 أيام، sel_days = 100 أيام
risk_percentage = min(int((35 ÷ 100) × 100) + 15, 95)
                = min(int(35) + 15, 95)
                = min(50, 95)
                = 50% (خطر متوسط-عالي)

مثال 3: P_var = 60 أيام، sel_days = 100 أيام
risk_percentage = min(int((60 ÷ 100) × 100) + 15, 95)
                = min(int(60) + 15, 95)
                = min(75, 95)
                = 75% (خطر عالي)
```

### 5️⃣ حساب تجاوز التكلفة المتوقع (Expected Cost Overrun)

```
cost_overrun_val = min(int((P_var ÷ sel_days) × 35), 100)
```

**الشرح التفصيلي:**
1. `P_var ÷ sel_days`: نسبة التأخير
2. `× 35`: معامل الأثر المالي (كل نسبة تأخير = 35% من التأثير على التكلفة)
3. `int(...)`: تحويل إلى عدد صحيح
4. `min(..., 100)`: حد أقصى 100% (لا يمكن تجاوز أكثر من الضعف)

**المنطق الاقتصادي:**
- كل يوم تأخير يسبب تكاليف إضافية:
  - رواتب الموظفين والعمالة
  - إيجار المعدات والآليات
  - إيجار الموقع (إن وجد)
  - المراقبة والإشراف
  - الفائدة على رأس المال المستثمر

**أمثلة عددية:**
```
مثال 1: P_var = 13.5 أيام، sel_days = 50 أيام
cost_overrun_val = min(int((13.5 ÷ 50) × 35), 100)
                 = min(int(9.45), 100)
                 = min(9, 100)
                 = 9% (تجاوز متوقع: 9%)
                 
إذا كانت الميزانية = 1,000,000 ريال
التكلفة الإضافية = 1,000,000 × 0.09 = 90,000 ريال

مثال 2: P_var = 35 أيام، sel_days = 100 أيام
cost_overrun_val = min(int((35 ÷ 100) × 35), 100)
                 = min(int(12.25), 100)
                 = min(12, 100)
                 = 12% (تجاوز متوقع: 12%)
```

## 📌 ملاحظات مهمة

1. **المعاملات المستخدمة قابلة للتعديل:**
   - معامل التأخير الأساسي: `0.4` (يمكن أن يزيد أو ينقص حسب نوع المشروع)
   - معامل الطقس: `0.15` (خاص بالمناطق الحارة)
   - هامش الأمان: `15` (يضاف لمعدل الخطر)
   - معامل التأثير المالي: `35` (نسبة الأثر على التكلفة)

2. **الحد الأقصى للخطر (95%):**
   - يمنع النظام من إعطاء تقييم متشائم جداً
   - حتى في أسوأ الحالات، لا يتجاوز معدل الخطر 95%

3. **تطبيق الصيغ الحسابية:**
   - جميع الحسابات آنية (Real-time)
   - تتغير تلقائياً عند تغيير أي متغير
   - تدعم اتخاذ قرارات سريعة
